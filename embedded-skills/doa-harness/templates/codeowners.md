# CODEOWNERS 模板

**多平台适配模式**：根据代码托管平台选择路径和语法格式。

---

## 平台路径选择

| 平台 | 文件路径 | 说明 |
|------|---------|------|
| GitHub | `CODEOWNERS`（项目根目录） | 也支持 `.github/CODEOWNERS` 和 `docs/CODEOWNERS` |
| GitLab | `CODEOWNERS`（项目根目录） | 支持 `[Section]` 分组语法 |
| Azure DevOps | 不生成文件 | 提示用户通过分支策略 → 自动审阅者规则配置 |

---

## 生成规则

扫描项目所有子项目目录，按以下映射自动标注 owner：

| 目录模式 | 默认 Owner | 说明 |
|----------|-----------|------|
| `core/` `kernel/` `framework/` `Framework/` | `{team_owner}` | 核心 / 底层框架 |
| `auth/` `security/` `Authorization/` `License/` | `{team_owner}` | 认证安全 |
| `infra/` `deploy/` `ActionFilter/` | `{team_owner}` | 基础设施 |
| `migrations/` `database/` `SQL_Script_BackUp/` | `{team_owner}` | 数据库变更 |
| `BaseEntity/` | `{team_owner}` | 基础实体 |
| `app/` `layouts/` (前端核心配置) | `{team_owner}` | 前端核心 |
| 入口文件: `Program.cs` `Startup.cs` | `{team_owner}` | 启动入口 |
| CI 配置文件 | `{team_owner}` | CI/CD |
| `.github/copilot-instructions.md` | `{team_owner}` | Agent 指令 |
| `.github/AGENTS.md` | `{team_owner}` | Agent 知识索引 |

`{team_owner}` 优先使用用户在 Step 2 提供的团队 Owner，未提供则用 `@todo-fill-team`。

---

## GitHub 输出模板

```
# CODEOWNERS — Generated by doa-harness v2.0 on {date}
# 修改核心目录时自动触发对应团队 review

{foreach sub in subprojects}
# {sub.name}
{foreach dir in sub.locked_dirs}
/{sub.path}/{dir}/                {team_owner}
{/foreach}
{/foreach}

# CI/CD
{foreach ci_file in ci_files}
/{ci_file}                        {team_owner}
{/foreach}

# Agent 配置
/.github/copilot-instructions.md  {team_owner}
/.github/AGENTS.md                {team_owner}
```

## GitLab 输出模板（带 Section 分组）

```
# CODEOWNERS — Generated by doa-harness v2.0 on {date}

{foreach sub in subprojects}
[{sub.name}]
{foreach dir in sub.locked_dirs}
/{sub.path}/{dir}/                {team_owner}
{/foreach}

{/foreach}
[CI/CD]
{foreach ci_file in ci_files}
/{ci_file}                        {team_owner}
{/foreach}

[Agent 配置]
/.github/copilot-instructions.md  {team_owner}
/.github/AGENTS.md                {team_owner}
```

## Azure DevOps 提示

当平台为 Azure DevOps 时，不生成 CODEOWNERS 文件，而是输出提示：

```
Azure DevOps 不支持原生 CODEOWNERS 文件。
请通过以下方式配置等效的代码审阅规则：
1. 项目设置 → 仓库 → 分支策略 → 自动包含的审阅者
2. 按路径筛选器添加审阅者规则
3. 对应的路径和审阅者：
{foreach dir in locked_dirs}
   路径: /{dir}/  →  审阅者: {team_owner}
{/foreach}
```

---

## 增量更新策略

**更新模式下**（`--update`）：
1. 读取已有 CODEOWNERS 文件
2. 扫描新增的核心目录
3. 仅追加新行，不修改已有 owner 映射
4. 更新日期标记
